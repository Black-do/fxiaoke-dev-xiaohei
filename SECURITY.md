# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

我们非常重视项目的安全性。如果你发现安全漏洞，请遵循以下步骤：

1. **不要**创建公开的 Issue
2. 发送邮件至 locc233@163.com，包含：
   - 漏洞描述
   - 复现步骤
   - 潜在影响
   - 建议的修复方案（如有）

### 预期响应时间

- 48 小时内确认收到报告
- 7 天内完成初步评估
- 定期更新修复进度
- 如愿意，在安全公告中致谢

## Security Best Practices

使用本项目时，请遵循以下安全最佳实践：

### 1. 依赖管理

- 定期更新依赖包
- 使用 `pip list --outdated` 检查过期依赖
- 关注依赖的安全公告

### 2. 代码安全

- 不要在代码中硬编码敏感信息
- 使用环境变量存储配置
- 定期审查代码变更

### 3. 文档安全

- 不要在文档中泄露 API 密钥或密码
- 使用示例数据而非真实数据
- 审查提交历史中的敏感信息

### 4. 开发环境

- 使用虚拟环境隔离项目依赖
- 不要将 `.env` 文件提交到版本控制
- 定期清理缓存文件

## Known Security Considerations

### API 文档安全

本项目包含纷享销客 API 文档，请注意：

- API 调用需要有效的认证令牌
- 不要在公开场合分享 API 密钥
- 遵守纷享销客的 API 使用条款

### 代码执行安全

`api_doc_helper.py` 会读取本地文件：

- 仅处理 `.md` 文件
- 不会执行外部代码
- 不会发送网络请求（除非使用外部向量搜索库）
